# Contributing to Bisto BinaryStorage

Thank you for considering contributing to the Bisto BinaryStorage project! We welcome all types of contributions, whether it's reporting bugs, suggesting features, or submitting code improvements.

## How to Contribute

### 1. Reporting Bugs
If you encounter a bug or issue with the library, please help us by submitting a detailed bug report. This will help us fix the issue faster and improve the overall project.

To submit a bug report:
- Search existing issues to ensure the problem hasn’t already been reported.
- Open a new issue with the following details:
  - **Steps to reproduce**: Provide a clear and concise description of the steps to replicate the issue.
  - **Expected behavior**: Describe what you expected to happen.
  - **Actual behavior**: Describe what actually happened.
  - **Environment details**: Include the version of Bisto BinaryStorage, .NET version, OS, and any other relevant details.

### 2. Suggesting Features
We are always open to suggestions for new features or improvements. To suggest a feature:
- Search existing issues and discussions to see if your idea has already been proposed.
- Open a new issue titled “Feature Request: [Your Feature Name]” with a clear description of the proposed feature, its use case, and any related details.
- If possible, provide examples of how the feature could be used in practice.

### 3. Code Contributions
We appreciate contributions from the community! Before submitting a pull request, please ensure the following:
- The contribution aligns with the project's goals and codebase.
- Your changes follow the coding standards and conventions used in the project.
- All code changes are covered by unit tests (if applicable).
- All existing and new tests pass.

#### Steps to Submit a Pull Request:
1. **Fork the repository**: Clone the project to your GitHub account.
2. **Create a new branch**: Use a meaningful name like `feature/my-feature` or `bugfix/issue-123`.
3. **Make your changes**: Add your code, tests, and any relevant documentation updates.
4. **Run the tests**: Ensure all tests pass.
5. **Commit your changes**: Use a descriptive commit message like `Added feature to support XYZ`.
6. **Push to your branch**: Push your changes to your fork on GitHub.
7. **Submit a pull request**: Navigate to the main repository and submit your pull request.

### 4. Documentation Contributions
Documentation is just as important as code! If you notice a typo, error, or outdated content in the documentation, feel free to submit corrections.
- Make sure your changes are clear and concise.
- For major documentation updates, open an issue first to discuss the proposed changes.

## Development Guidelines

### Code Style
Please follow these general coding guidelines:
- **Consistency**: Follow the established code style (naming conventions, formatting, etc.) used in the repository.
- **Comments**: Ensure your code is well-documented with comments where necessary.
- **Tests**: All new features or bug fixes should have associated unit or integration tests.

### Unit Tests
- Run all existing tests to ensure your changes don’t introduce regressions.
- Add new tests to cover any new functionality or bug fixes.

### Code Reviews
- All pull requests will go through a code review process. The review will check for code quality, consistency, and adherence to project standards.
- Be prepared to make changes based on feedback from reviewers.

## Getting Help
If you need help or have questions about contributing, feel free to:
- Open an issue for general inquiries.
- Start a discussion in our GitHub Discussions page.

Thank you again for your contributions! We value your time and effort to make Bisto BinaryStorage better for everyone.
